# Sessão 2026-03-21 — Mudanças e Próximos Passos

## Contexto

Continuação da sessão anterior. O banco MySQL já estava rodando via Docker com o schema criado. Esta sessão finalizou a migração de questões, corrigiu bugs legados no backend e migrou o frontend para o novo schema normalizado.

---

## Mudanças realizadas

### Backend

#### `statl/__init__.py`
- Trocado driver `mysql+mysqlconnector` → `mysql+pymysql` (mysql-connector-repackaged é incompatível com Python 3.13)
- Removido `print` de debug do `create_app`
- Corrigido import dos models para evitar shadowing do blueprint `questions`

#### `statl/services/auth_service.py`
- **Bug crítico corrigido:** `reset_password` chamava `get_user_by_email(user_id)` passando um ID no lugar de email — agora chama `get_user_by_id(user_id)`
- Removido `print` de debug do `login_user`

#### `statl/routes/auth.py`
- Removidos imports desnecessários: `decode_token`, `current_user`, `select`, `ResetPasswordRequestForm`, `get_user_by_email_service`
- `POST /login` não aceita mais `GET`

#### `statl/migrate_questoes.py`
- Corrigido `UnicodeEncodeError` no Windows: removido caractere `→` dos prints

#### `requirements.txt`
- Substituído `mysql_connector_repackaged==0.3.1` por `pymysql==1.1.2`

### Frontend

#### `app/(app)/QuizInProgressScreen.tsx`
- **Refatoração completa:** migrado de `/questions/rand/:num` para `/questions/filtered`
- Estado mudou de arrays paralelos (`issues[]`, `answers[][]`, `correct_answers[]`) para `QuizQuestion[]` com shape normalizado
- Alternativas renderizadas de `current.alternatives` em vez de `answer_a`–`answer_e`
- Suporte a parâmetros de filtro: `chapter_id` e `difficulty` lidos do router
- Checagem de resposta continua client-side

#### `app/components/CustomAccordion.tsx`
- Novos props: `chapters`, `chapterId`, `setChapterId`, `difficulty`, `setDifficulty`
- Adicionados seletores de capítulo (botões "Todos / Cap. 1 / Cap. 2 ...") e dificuldade (Todas / Fácil / Médio / Difícil)
- Largura aumentada de 40% para 80%

#### `app/(tabs)/questions.tsx`
- Busca capítulos de `/questions/chapters` no mount
- Passa `chapter_id` e `difficulty` como params para `QuizInProgressScreen`

#### `app/services/CheckAnswer.tsx`
- Removido `export default` (causava crash no Metro — Expo Router trata arquivos dentro de `app/` como rotas e rejeita async functions como default export)
- Contrato atualizado para novo API: `{ question_id, answer }` em vez de `{ question_index, answer }`
