<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| professor frontend -> `/lists` draft/publish routes | UI state can become stale or double-submit if the screen does not refetch after picker/save transitions |
| shared picker route -> question management actions | A shared component must not accidentally expose admin-only destructive affordances in pick mode |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Tampering | CreateNewList stale draft state | mitigate | Always refetch `GET /lists/<id>` after returning from the picker and after publish/save responses |
| T-04-06 | Elevation of Privilege | SharedQuestionViewer | mitigate | `mode="pick"` must suppress edit/delete actions and expose only selection actions |
| T-04-07 | Repudiation | Published edits | mitigate | UI surfaces the backend `change_log` block directly on the authoring screen so edits are never silent |
</threat_model>
