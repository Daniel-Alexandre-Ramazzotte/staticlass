## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip row inline gap (`gap="$2"` = 8px in Tamagui, use `gap="$1"` for tightest chips) |
| sm | 8px | Gap between chips in XStack rows (`gap="$2"`) |
| md | 16px | Section vertical spacing within accordion content (`gap="$4"`) |
| lg | 24px | Padding inside accordion content panel (`padding="$4"`) |
| xl | 32px | Vertical scroll content padding |
| 2xl | 48px | Header top padding (pt="$8" in Tamagui = ~32px; safe area start) |
| 3xl | 64px | Page-level bottom padding for scroll content |

Exceptions:
- Touch targets for filter chips use Tamagui `size="$2"` (~36px height) — acceptable for secondary filter controls. The primary form CTA button uses `size="$5"` (~44px) for minimum touch target compliance.
- The source dropdown/picker in AddNewQuestion must maintain a minimum tap height of 44px (matching existing `BotaoSeletor` buttons at `size="$3"`).

Source: layout constants from `app/frontend/src/constants/layout.ts` (`btnH = 44` on mobile)

### 2. QuestionsManager — source filter chip row (professor, admin roles only)

**File:** `app/frontend/app/(professor)/QuestionsManager.tsx`

This filter is **client-side only** (CONTEXT.md D-07). Add a `sources: string[]` state and a source chip row inside the `role === 'admin'` filter block, placed after the difficulty chip row. Use the identical Button pattern already used for difficulty:

```tsx
{FONTES_MANAGER.map((src) => (
  <Button
    key={src.value}
    size="$2"
    backgroundColor={sources.includes(src.value) ? palette.primaryBlue : palette.darkBlue}
    color={palette.offWhite}
    onPress={() => setSources((ss) => toggleS(ss, src.value))}
  >
    {src.label}
  </Button>
))}
{sources.length > 0 && (
  <Button size="$2" backgroundColor={palette.red} color={palette.offWhite}
    onPress={() => setSources([])}>
    ✕ Fonte
  </Button>
)}
```

Define `FONTES_MANAGER` at module scope with the same 7 values and order as the FONTES constant above.

Add `toggleS` helper (already exists in CustomAccordion; duplicate locally in QuestionsManager):
```ts
function toggleS(arr: string[], val: string): string[] {
  return arr.includes(val) ? arr.filter((x) => x !== val) : [...arr, val];
}
```

Extend `filteredQuestions` useMemo to include source filtering:
```ts
const filteredQuestions = useMemo(() => {
  let result = questions;
  if (sources.length > 0) {
    result = result.filter((q) => sources.includes((q as any).source));
  }
  // existing search filter...
  const normalizedSearch = searchValue.trim().toLowerCase();
  if (!normalizedSearch) return result;
  return result.filter((q) => {
    const title = (q.issue || q.enunciado || '').toLowerCase();
    return title.includes(normalizedSearch);
  });
}, [questions, searchValue, sources]);
```

Note: `QuestionListItem` type must gain `source?: string | null` field to support the filter. The API response from `/questions/professor/:id` will return `source` after Phase 2 backend work.

Expanded question detail card: add a `Fonte` line below `Seção`:
```tsx
<Text color="#24506e">
  {`Fonte: ${selectedQuestion.source || 'Não informada'}`}
</Text>
```
(Source: CONTEXT.md Claude's Discretion — "Whether to add source to the question detail card" → yes, include it for consistency with other metadata fields.)

### 3. AddNewQuestion — source dropdown field

**File:** `app/frontend/app/(professor)/AddNewQuestion.tsx`

Add a `fonte` state (`useState<string>('')`) and a source selector placed between the `Dificuldade` selector row and the `Resposta correta` selector row.

UI element: a row of `BotaoSeletor`-style buttons (same as difficulty row) rather than a native Picker, to stay consistent with the existing selection pattern for small option sets. 7 options will not fit in a single XStack row — use two XStack rows or `flexWrap="wrap"`.

Recommended implementation — wrap in a single `XStack flexWrap="wrap" gap="$2"` with `width="94%" alignSelf="flex-end"`:

```tsx
<YStack width="100%" gap={0}>
  <RotuloLabel>Fonte:</RotuloLabel>
  <XStack width="94%" alignSelf="flex-end" marginTop="$1" flexWrap="wrap" gap="$2">
    {FONTES_ADD.map(({ value, label }) => (
      <BotaoSeletor
        key={value}
        label={label}
        ativo={fonte === value}
        onPress={() => setFonte(value)}
      />
    ))}
  </XStack>
</YStack>
```

The field is optional (CONTEXT.md D-04): if left blank (`fonte === ''`), the backend defaults to `lista`. Do NOT add frontend validation for this field — it mirrors the existing optional `dificuldade` field pattern.

Include `fonte` in the API payload only when non-empty:
```ts
const payload = {
  // ...existing fields...
  source: fonte || undefined,
};
```

Define `FONTES_ADD` at module scope (same 7 values and order as above).

`BotaoSeletor` uses `flex={1}` which causes even distribution. With 7 items in a flex-wrap row, set `flex={undefined}` and `minWidth="28%"` to prevent single-item rows from stretching full width. This is the only deviation from the existing BotaoSeletor pattern.
