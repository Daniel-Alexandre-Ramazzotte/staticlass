## Spacing Scale

Declared values (Tamagui token equivalents mapped to pixel values):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`$1`) | Icon gaps, inline padding between text elements |
| sm | 8px (`$2`) | Compact row gaps, label-to-value gap inside stat cards |
| md | 16px (`$4`) | Default horizontal screen padding, card internal padding |
| lg | 24px (`$6`) | Section vertical gaps, outer horizontal padding for action areas |
| xl | 32px (`$8`) | Major layout breaks between header and body |
| 2xl | 48px | Safe-area header top padding (`pt="$10"` = ~40px; treat as 48px intent) |
| 3xl | 64px | Reserved for page-level spacing (not used in this phase) |

Exceptions:
- Ranking row height: minimum 52px touch target per row (WCAG 2.5.5 — meets 44px minimum with margin).
- Pinned own-row bottom inset: 0px (flush to bottom safe area); internal padding follows md (16px).
- Streak day dots in WeekRow: 36px wide × 20px tall pill — existing shape from `(tabs)/profile.tsx`, do not change.
- XP bar track height: 8px (existing in `profile.tsx`, retain).

## Typography

All sizes use Tamagui `Text` component. Weights expressed as `fontWeight` string values.

**Declared sizes (exactly 4):** 22px, 20px, 15px, 12px.

The existing 56px score display on ResultScreen is pre-existing, not introduced by this phase, and is therefore not included in the phase type scale. Do not modify it.

| Role | Size | Weight | Line Height | Font Family | Usage |
|------|------|--------|-------------|-------------|-------|
| Heading | 22px | bold (700) | 1.2 | primaryFontA (ChauPhilomeneOne) | Screen titles (Statistics, Ranking header) |
| Subheading | 20px | bold (700) | 1.2 | default (system) | XP earned display on ResultScreen (`+{xpTotal} XP`), streak count large display on profile header |
| Body | 15px | 400 | 1.5 | primaryFontC (Carlito) | Ranking row names, XP values in list; use bold (700) weight at this size for rank position number (e.g. `#12`) instead of a separate size |
| Label | 12px | 400 | 1.4 | default | Sub-labels, column headers (e.g. `XP`), opacity 0.7–0.75 on dark backgrounds |

Size mapping applied to collapse all extras into the 4-size scale:

| Previous mention | Mapped to | Rationale |
|-----------------|-----------|-----------|
| 13px (ResultScreen multiplier line) | 12px Label | Decorative sub-label role |
| 14px (RankingRow rank number, XpStreakHeader label) | 15px Body | Body role, use bold weight for visual distinction |
| 16px (ranking position, RankingRow XP column) | 15px Body bold | Same body role; bold weight provides distinction without new size |
| 20px (XP earned display, streak count display) | 20px Subheading | Promoted to explicit Subheading role in the 4-size scale |

Sources: codebase scan of `ResultScreen.tsx`, `Statistics.tsx`, `profile.tsx`, `home.tsx`.

## Color

Source: `src/constants/style.tsx` palette + CONTEXT.md code insights.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#fcfcfc` (offWhite) | Screen backgrounds where content scrolls over light surface (Statistics pattern) |
| Secondary (30%) | `#093d60` (darkBlue) | Cards containing gamification data (streak calendar card, XP card) |
| Header surface | `#0074c3` (primaryBlue) | Screen header bars — consistent with all existing screens |
| Accent (10%) | `#55bf44` (primaryGreen) | See reserved list below |
| Destructive | `#f65151` (red) | Streak break indicator (red pill in WeekRow), back/exit buttons |

Accent (`#55bf44`) reserved for:
1. Streak day dot — practice day fulfilled (green pill)
2. XP earned value on ResultScreen ("+75 XP" number)
3. Streak count number on profile header
4. "Ver ranking" / navigation CTA button background (one button only per screen)
5. Trophy icons already rendered via `@tamagui/lucide-icons` Trophy component on profile

Accent must NOT be used on: ranking rows in general, text bodies, borders, backgrounds of non-CTA elements.

XP bar fill color: `#FFD700` (gold) — already in `profile.tsx`, retain. This is the only yellow used.

Pinned own-row in ranking: background `rgba(255,215,0,0.15)` (gold tint at 15% opacity) on a `palette.darkBlue` list surface, left border 3px solid `#FFD700`. This distinguishes the user's own row without requiring a new color.

## Screen-by-Screen Interaction Contract

### Profile Screen (aluno) — additions

Trigger: `useFocusEffect` refetch on every screen focus (existing pattern from Statistics.tsx).

New data needed beyond existing `score`:
- `streak` (int) — days in current streak
- `rank_position` (int) — student's global rank
- `last_practice_date` (date string) — to determine if today has been practiced (green vs. pending dot for today)

Weekly calendar: currently hardcoded mock. This phase wires it to real data from `GET /users/historico` (already available). Map `criado_em` dates to the 7-day row; today's date is the rightmost dot in the current week. If `criado_em` for today exists → green; if today has no entry → `null` (grey, pending). Past days without entry → red (false).

No new navigation added to profile; Ranking screen access via existing tab bar entry in `(app)/Ranking.tsx`.

### Ranking Screen

**Focal point declaration:**
- On initial load (first visit or cold load): the #1 ranked row is the primary visual anchor. It sits at the top of the list, with rank number `#1` in 15px bold white. The user's eye travels top-to-bottom naturally; the top row carries implicit status weight.
- On returning visits (screen already loaded, user scrolls back up): the pinned own-row at the bottom is the primary anchor for the returning user. It is always visible regardless of scroll position and uses the gold tint background (`rgba(255,215,0,0.15)`) + left border (`#FFD700`) to draw attention to the user's own standing. This dual-anchor pattern mirrors the Duolingo leaderboard: aspirational anchor (#1) on load, personal anchor (own row) for returning motivation.

Navigation: already reachable from tab bar (stub exists). No new navigation wiring needed.

Scroll behavior: FlatList with `onEndReached` callback loading next 20 items. "Carregar mais" button as fallback at list tail if automatic pagination is not implemented.

Own-row pinning: render pinned row as a separate `YStack` below the FlatList, outside scroll, above the bottom safe area. Always visible regardless of scroll position.

Pull-to-refresh: implement `RefreshControl` with `refreshing` state, color=`palette.primaryGreen`. Standard pattern.

### ResultScreen — XP feedback moment

After `POST /gamification/record-session` resolves (or `/users/salvar-resultado` is migrated):
- Replace the static `+{acertos*10} pts` text with the actual `xp_gained` value from response.
- If the API call is still in-flight when the screen mounts, show `"Calculando XP..."` in opacity 0.6, then update when resolved.
- If API call fails: show `"+{acertos*10} XP"` (client-calculated fallback, no multiplier disclosed) — avoids blank.

## States Required Per Screen

### Ranking Screen

| State | Trigger | Visual |
|-------|---------|--------|
| Loading | initial mount | ActivityIndicator centered, header still visible |
| Loaded | API success | FlatList with rows + pinned own-row |
| Load-more loading | scroll end or button tap | ActivityIndicator inline at list bottom, rows stay visible |
| Empty | API returns 0 items | Centered heading + body copy (see Copywriting) |
| Error | API returns non-2xx | Centered error text color=`palette.red` |
| Refreshing | pull-to-refresh | RefreshControl spinner color=`palette.primaryGreen` |

### ResultScreen XP block

| State | Trigger | Visual |
|-------|---------|--------|
| Calculating | API in-flight | `"Calculando XP..."` opacity 0.6, no multiplier line |
| Resolved | API 2xx | `"+{N} XP"` color=primaryGreen + multiplier line if streak >= 3 |
| Failed | API error | `"+{client_calc} XP"` — client-side fallback, no multiplier |

### Profile aluno XP/streak block

| State | Trigger | Visual |
|-------|---------|--------|
| Loading | focus event, data in-flight | Rank shows `—`, streak shows `—` |
| Loaded | API success | Full XP bar, streak count, rank position |
| Error | API error | Retain last known values (do not blank screen) |
