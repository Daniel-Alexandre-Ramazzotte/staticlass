```typescript
if (response?.status !== 201) {

    response?.data.error || 'Erro ao registrar. Tente novamente.'  // BUG: .data.error throws if data is undefined
  );

Root cause: When Axios throws with no server response (network failure, DNS failure, CORS blocked),
`error.response` is `undefined`. The caller reads `response?.data.error` — the `?.` protects `.data`
access from undefined `response`, but `?.data.error` (without the second `?`) throws if `data` is
defined but `error` is not a key on it. More importantly, when `response` is `undefined`, the
condition `response?.status !== 201` evaluates to `true` (undefined !== 201), and `response?.data`
evaluates to `undefined`, so `response?.data.error` throws a TypeError.

import api from './api';

const RegisterNewUserService = async (data: {
  email: string;
  password: string;
  confirm_password: string;
  name: string;
}) => {
  console.log(data);
  try {
    const response = await api.post('/auth/register', data);
    return response;
  } catch (error: any) {
    console.error('Error:', error);
    return error.response ?? { status: 0, data: { error: 'Sem conexão com o servidor.' } };
  }
};
export default RegisterNewUserService;
```

Key change: `return error.response` → `return error.response ?? { status: 0, data: { error: 'Sem conexão com o servidor.' } }`

The `??` operator returns the right-hand side only when the left-hand side is `null` or `undefined`.
When the server returns a real 4xx, `error.response` is the Axios response object and is returned as-is.
When there is no server response, the synthetic object is returned, giving the caller a well-typed
object with `.status === 0` and `.data.error` set to a Portuguese user-facing message.
