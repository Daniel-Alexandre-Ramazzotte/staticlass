<objective>
Replace the hardcoded `http://localhost:5000` URL in the password reset email with a configurable
environment variable `APP_BASE_URL`.

Purpose: The current code sends a reset link pointing to `localhost:5000` regardless of deployment
environment. This link is broken in any non-local environment — clicking it from a real email opens
nothing. The fix makes the base URL configurable via `.env` so production deployments use the
correct domain without code changes.

Note on deep-link design: This fix makes the URL configurable. Whether the link should be an HTTP
URL (to a web page) or a mobile deep link (`staticlass://reset-password?token=...`) is a separate
design question flagged in the research file. Plan 01-02 scopes the fix to the env var change only.

Output: Modified `email_service.py` + updated `.env` with the `APP_BASE_URL` key.
</objective>

```python
        subject=subject,
        recipients=[to],
        body=body
    )
    print(current_app.config["MAIL_SERVER"])
    print(current_app.config["MAIL_PORT"])
    with current_app.app_context():
        mail.send(msg)
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Backend → Email provider (SMTP) | Token embedded in email body sent to external SMTP server |
| Email → User's mail client | Reset link in email body; user clicks it to initiate reset |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-02-01 | Information Disclosure | send_reset_email — token in URL query param | accept | Token is short-lived (itsdangerous TimedSerializer); token expiry enforced by `verify_reset_token`; HTTPS in production protects token in transit |
| T-01-02-02 | Tampering | APP_BASE_URL env var — redirect manipulation | accept | This env var is server-side only; an attacker cannot set it without server access; if server is compromised, this is not the threat vector |
| T-01-02-03 | Spoofing | Password reset email — phishing vector | accept | Email is sent FROM a configured address; recipient can inspect headers; no additional mitigation needed at this layer (DKIM/SPF is infra config) |
| T-01-02-04 | Information Disclosure | .env file committed to git | mitigate | Verify .env is in .gitignore before committing; never commit the actual secret values; only APP_BASE_URL=http://localhost:5000 (not a secret) appears in .env |
</threat_model>
