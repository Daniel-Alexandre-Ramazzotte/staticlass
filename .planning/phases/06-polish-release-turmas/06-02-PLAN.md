    def verify_email_token_json():
        data = request.get_json() or {}

        if not token:
            return jsonify({"error": "Token é obrigatório"}), 400

        if not user:

            return
        try:
            token = generate_verification_token(user.id)
            send_verification_email(to=email, token=token)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| email link → app | Token arrives via deep link query param; must be validated server-side, not trusted client-side |
| POST /auth/resend-verification | Public endpoint — must not reveal whether email exists |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-02-01 | Information Disclosure | resend-verification endpoint | mitigate | Always return 200 regardless of whether email exists (already in action text) |
| T-06-02-02 | Elevation of Privilege | verify-email-token | mitigate | Token validated server-side by `verify_verification_token` with 24h expiry and HMAC signature |
| T-06-02-03 | Tampering | deep link token param | accept | Token is HMAC-signed by itsdangerous; forged tokens rejected by `verify_verification_token` |
| T-06-02-04 | Denial of Service | resend-verification spam | accept | Low-severity for v1 internal release; rate limiting deferred to backlog |
</threat_model>
